## Why AutoGit?

Developers shouldn't have to repeat the same commands dozens of times a day.

AutoGit intelligently automates the most common Git workflow:

```text
Code → Commit → Push
```

with a single command.

```bash
autogit
```

No unnecessary typing. No remembering branch names. No repetitive commands.

Just build.

---

## ✨ Features

* **Automatic branch detection**
* **Smart change detection**
* **Displays modified files before committing**
* **One-command add → commit → push**
* **Beautiful colored CLI output**
* **Cross-platform support**
* **Zero configuration**
* **Fast and lightweight**

```bash
git clone https://github.com/yourusername/autogit.git

cd autogit

pip install -r requirements.txt
```

---

## Usage

Inside any Git repository:

```bash
autogit
```

AutoGit automatically:

1. Detects the current branch.
2. Checks whether there are changes.
3. Shows modified files.
4. Creates a commit.
5. Pushes to the correct remote branch.

---

## Philosophy

AutoGit follows a simple principle:

> **Tools should disappear into the background.**

Git is powerful, but everyday workflows shouldn't require repetitive commands.

AutoGit aims to make version control feel effortless.
